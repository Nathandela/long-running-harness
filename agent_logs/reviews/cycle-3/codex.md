  Clearing `queued` in the outer `catch` avoids replaying stale state, but it also discards newer work when the failure happens inside `processQueue()`. The new test only covers failure of the first inflight save, not failure during queued processing. Preserve or explicitly retry the newest queued snapshot on queued-save failure, and add a regression test for that case.
